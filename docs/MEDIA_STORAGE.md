# Медиа: изображения, фото, видео

Решение по хранению пользовательских медиа (обложки статей, фото объявлений,
рекламные баннеры) и спецификация водяного знака.

## Решение

Гибрид по типу медиа — не «или-или»:

| Медиа | Где хранится | Почему |
|---|---|---|
| **Фото / изображения** | у нас — объектное хранилище + CDN, с серверной обработкой | дёшево, переживает блокировки, работает для нетехнического пользователя, позволяет модерацию и снятие EXIF |
| **Видео** | встраивание с площадок (YouTube/Vimeo/Rutube) по белому списку `embed_url` | транскодинг + egress-трафик своего видео разорят бюджет; у площадок свой CDN |
| **Чужие ссылки** | принимаем как импорт, но **сразу перекачиваем файл к себе** (fetch-and-rehost) | лечит link rot, hotlink-защиту, утечку IP читателя и блокировку чужого хоста |

Почему не «только ссылки на облака пользователей»: они ломаются при блокировке
foreign-хостов в РК (прямое противоречие миссии), недоступны нетехническому
пользователю (нет прямого URL с телефона), гниют (link rot), не модерируются и
утекают IP читателя третьим сторонам. Экономия на хранении мнимая.

Почему не локальный диск приложения как основное: не масштабируется, единая
точка отказа, дорогая отдача с origin. Объектное хранилище — тот же «у нас», но
без этих минусов.

## Реализация — стадии

**Media-1 (сделано):** пакет `pkg/modules/media`.
- Интерфейс `Store` (`Put/Delete/URL`) + бэкенд `FSStore` (локальный диск,
  раздача с `PublicPrefix`). Ключ объекта = SHA-256 контента, шардинг по 2 hex.
- Конвейер `processImage`: декод (jpeg/png/gif/webp) → флэттен на белый фон →
  ресайз до `MaxDimension` (CatmullRom) → **водяной знак** → энкод JPEG q82.
  Ре-энкод из сырых пикселей **срезает EXIF (в т.ч. GPS)** — безопасность автора.
- Эндпоинт `POST /media/upload` (cookie-сессия, 401 для гостя, 415 для не-картинки,
  лимит `MaxUploadBytes`). Формы редактора статьи и подачи объявления получили
  кнопку «Загрузить фото» (прогрессивное улучшение поверх поля URL).
- Конфиг `media.*` (`internal/config`): `backend`, `dir`, `public_prefix`,
  `max_upload_bytes`, `max_dimension`, `watermark`. Дефолты — рабочие «из коробки».

**Media-2 (потом):** бэкенд `S3Store` (S3-совместимый / MinIO) за тем же
интерфейсом; приоритет — провайдер с **нулевой платой за egress** через CDN,
чтобы трафик фото не бил по бюджету. Подписанные прямые загрузки (минуя наш
сервер). Fetch-and-rehost для импорта по ссылке. AVIF/WebP-энкод. Коррекция
ориентации по EXIF до его удаления. Поле `embed_url` для видео + oEmbed-рендер.

## Водяной знак

- Источник: `web/static/brand/shanraq-mark.svg` — **статичный** инлайн-вариант
  логотипа (исходный `shanraq.svg` стилизован через CSS-классы и keyframes,
  которые растеризатор не рендерит и которые не должны анимироваться на знаке).
- Размер: **иконка 32px**, правый верхний угол, отступ 12px.
- **Полупрозрачный** (alpha ≈ 140/255) — виден как бренд-штамп, не перекрывает фото.
- Без анимации (одиночный статичный оверлей). Рендер SVG в 4× с последующим
  уменьшением — чтобы толстые штрихи логотипа не сливались в пятно на 32px.
- Настраивается: `media.watermark: false` отключает.
- Видео: на своём (когда появится) — тот же знак фильтром overlay в ffmpeg;
  встроенное чужое видео мы водяным знаком не помечаем (не наш плеер).
